# ⚓ ПЛАН РАЗРАБОТКИ: Anchor Watch Clone (Android & iOS)
**Кроссплатформенное приложение для отслеживания якорной стоянки**  
**Срок MVP:** 8 недель | **Стек:** Flutter + RevenueCat + Native Modules  
**Цель:** Воспроизвести ключевой функционал Anchor Watch с кроссплатформенной архитектурой, соблюдая приватность и нативный UX.

## 🔐 РЕАЛИЗАЦИЯ ПРИВАТНОСТИ И REMOTE WATCH

```
🔹 Локальное хранение (по умолчанию):
• Все данные (якоря, история, настройки) — в зашифрованной SQLite (sqlcipher)
• Ключ шифрования — в flutter_secure_storage (Keychain/Keystore)
• Нет аналитики, нет трекеров, нет внешних запросов без согласия

🔹 Remote Watch (опционально, post-MVP):
• Абстракция SyncProvider с двумя реализациями:
  - ICloudSync (Swift native module): CloudKit private database
  - FirebaseSync (Flutter): Firestore с E2E-шифрованием (ключ на устройстве)
• Перед отправкой: данные шифруются (encrypt + RSA/AES), расшифровка только на устройстве пользователя
• Пользователь сам включает синхронизацию и выбирает доверенные устройства
• В настройках: "Что синхронизируется", "Удалить данные с сервера", "Экспорт ключа"

🔹 Прозрачность:
• Экран "Приватность" с простым языком: "Мы не видим ваши данные"
• Кнопка "Проверить шифрование" — технически подкованным пользователям
• Открытый исходный код модулей шифрования (опционально)
```

## 💡 СОВЕТЫ РАЗРАБОТЧИКУ

1. **Подайте на Critical Alerts заранее** — модерация Apple может занять 2–4 недели. Готовьте fallback.
2. **Сделайте экран "Проверка тревоги"** — пользователь должен сам протестировать звук/вибрацию до реальной стоянки.
3. **Добавьте авто-диагностику прав** — при запуске проверять, включены ли нужные разрешения, и показывать пошаговый гайд.
4. **Не гонитесь за идеальной точностью** — телефонный GPS имеет погрешность 3–10 м. Алгоритм должен учитывать это через `accuracy`-маржу.
5. **Тестируйте на "убийцах фоновых процессов"** — особенно китайские и корейские бренды (Xiaomi, Samsung, Huawei).
